Git安全性配置

发表于 2014-05-31   |   分类于 Git   |  

Git的好处是大家都知道的，最强版本控制，没有之一，网上相关资料很多，这里总结一下安装好Git之后需要进行的安全配置，然后才能成功地对线上代码仓库进行操作。

下载客户端

Mac用户比较方便，Git已经在终端(Terminal)集成了，Windows用户需要到官网进行下载。

  • Windows系统用户下载后若不配置环境变量，可直接打开开始->Git->Git Bash使用控制台的Git。
  • 其他系统打开终端即可。

将SSH Key设置到Github

本机SSH Key生成完毕后，需要添加到Github上，完成最终的连接设置。

用文本编辑器或记事本打开id_rsa.pub文件（打开Finder直接跳转到路径~/.ssh），并准确地复制全部内容。（如果看不到这个文件，请显示隐藏文件）

打开Github并登陆，在网页的右上角点击扳手图标（Account Settings）

选择SSH Keys项，把复制的内容粘贴到Key一栏，Title留空，然后点击Add Key按钮即可。

PS：如果需要配置多个GitHub账号，可以参看这个多个github帐号的SSH key切换，不过需要提醒一下的是，如果你只是通过这篇文章中所述配置了Host，那么你多个账号下面的提交用户会是一个人，所以需要通过命令git config –global –unset user.email删除用户账户设置，在每一个repo下面使用git config –local user.email ‘你的github邮箱@mail.com’ 命令单独设置用户账户信息
